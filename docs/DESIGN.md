## 0. 设计风格更新 (v2.0 → v2.1)

### 0.1 新设计风格: BMW 风格

MarkdownHub 前端已全面重构为 **BMW** 风格——一个源自德国汽车工业的精确、硬朗的设计语言。核心特点：

- **BMW Blue 强调色** - 品牌标志性蓝色 #1c69d4 作为唯一强调色
- **零圆角设计** - 所有元素使用直角矩形，棱角分明
- **大写字母** - 按钮和标题使用 uppercase + 紧凑字间距
- **深色 Hero 区** - 黑色背景配白色大写文字

### 0.2 BMW 设计哲学

> "BMW's design language is precision engineering made visual — every surface, every line, every proportion speaks of German engineering excellence."

**核心原则**:
- 零圆角 - 直角代替所有圆角，传达精确和果断
- BMW Blue - 唯一允许的彩色，用于焦点状态和强调
- 紧凑字间距 - uppercase 标题 + letter-spacing: 0.05em-0.15em
- 深色对比 - Hero 区域使用近黑背景配白字

## 2. 设计理念与原则

### 1.1 设计愿景

MarkdownHub 的设计追求**专业、高效、沉浸**的编辑体验。作为面向研发团队和技术写作者的协作工具，界面应当：

- **聚焦内容**：减少视觉干扰，让用户专注于 Markdown 写作本身
- **清晰层次**：通过一致的视觉语言建立信息层次，降低认知负担
- **高效协作**：实时协作状态直观可见，权限控制一目了然
- **优雅克制**：采用中性配色，避免喧宾夺主，让文档内容成为主角

### 1.2 设计原则

| 原则 | 描述 | 实践示例 |
|------|------|----------|
| **内容优先** | 界面服务于内容，而非内容服务于界面 | 编辑器最大化空间，工具栏按需隐藏 |
| **一致性** | 相同的操作产生相同的结果 | 统一的图标、间距、动效 |
| **可预测性** | 用户应能准确预测操作结果 | 拖拽有视觉反馈，删除有确认 |
| **容错性** | 允许撤销，提供安全网 | 所有危险操作可撤销或确认 |
| **效率优先** | 常用操作一步到位 | 快捷键支持，键盘导航 |
| **无障碍** | 考虑不同能力的用户 | WCAG 2.1 AA 标准 |

### 2.1 色彩系统 (BMW Style)

#### 2.1.1 CSS 变量定义

```css
:root {
  /* BMW 调色板 */
  --bmw-blue: #1c69d4;      /* BMW Blue - primary accent */
  --near-black: #262626;     /* Primary text */
  --white: #ffffff;          /* Primary background */
  --meta-gray: #757575;      /* Secondary text, metadata */
  --surface-gray: #f5f5f5;  /* Card/surface background */
  --dark-surface: #1a1a1a;  /* Dark surface */
  --hero-black: #0a0a0a;    /* Hero background */

  /* Zero radius - BMW key characteristic */
  --radius: 0;
  --radius-sm: 0;
  --radius-md: 0;

  /* Surfaces */
  --bg: var(--white);
  --surface: var(--white);
  --surface-raised: var(--surface-gray);
  --surface-sunken: var(--light-gray);

  /* Text */
  --text-primary: var(--near-black);
  --text-secondary: var(--meta-gray);
  --text-muted: #999999;
}
```

### 2.5 阴影系统 (BMW 简约阴影)

**BMW 风格使用极简阴影** - 通过对比度和直角设计创造深度感，而非复杂阴影层次。

```
阴影规格表 (BMW 3-Level System):
┌─────────────────────────────────────────────────────────────────┐
│ 名称     │ 数值                                          │ 使用 │
├──────────┼────────────────────────────────────────────────┼──────┤
│ sm       │ 0 1px 2px rgba\(0,0,0,0.05\)                   │ 浮动元素 │
│ md       │ 0 2px 4px rgba\(0,0,0,0.08\)                  │ 卡片、按钮 │
│ lg       │ 0 4px 8px rgba\(0,0,0,0.1\)                   │ 下拉菜单 │
└──────────┴────────────────────────────────────────────────┴──────┘

BMW 阴影哲学:
- 简约为主 - 少即是多，避免过度装饰
- 深度通过对比 - 浅色背景上的微妙阴影
- 零圆角 + 轻阴影 = 精确感
```

#### 2.6.4 实际实现的动画

```css
/* 实际使用的过渡时长 */
transition: all 0.2s ease;           /* 常规过渡 */
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);  /* 按钮悬停 */

/* 实际使用的关键帧动画 */
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.6; }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

@keyframes toast-slide-in {
  from { transform: translateX(100%); opacity: 0; }
  to { transform: translateX(0); opacity: 1; }
}

@keyframes skeleton-loading {
  0% { background-position: -200px 0; }
  100% { background-position: calc(200px + 100%) 0; }
}
```

## 9. 无障碍设计 (Accessibility)

### 9.1 色彩对比

```
文字类型        │ 最小对比度 │ 使用场景
────────────────┼───────────┼────────────────────
正文文字        │ 4.5:1     │ 主要内容
大号文字 (≥18px)│ 3:1       │ 标题
组件边缘        │ 3:1       │ 输入框、按钮边框
装饰性内容      │ 无要求    │ 图标、装饰线
```

### 9.2 键盘导航

| 元素 | Tab 顺序 | 方向键 | 其他 |
|------|----------|--------|------|
| 按钮 | 1 | - | Enter/Space 激活 |
| 链接 | 1 | - | Enter 激活 |
| 输入框 | 1 | - | - |
| 下拉菜单 | 1 | ↑↓ 选择 | Enter 确认 |
| 树形列表 | 1 | ↑↓ 移动 | ← 折叠，→ 展开 |
| 模态框 | 1 | Tab 循环 | ESC 关闭 |
| Tab | 1 | ←→ 切换 | - |

### 9.3 ARIA 规范

```html
<!-- 按钮 -->
<button aria-label="关闭对话框">×</button>

<!-- 输入框 -->
<input
  type="text"
  aria-describedby="helper-id"
  aria-invalid="true"
  aria-errormessage="error-id"
/>

<!-- 模态框 -->
<div role="dialog" aria-modal="true" aria-labelledby="title-id">
  <h2 id="title-id">标题</h2>
</div>

<!-- 树形 -->
<ul role="tree" aria-label="文档结构">
  <li role="treeitem" aria-expanded="true">
    父节点
    <ul role="group">
      <li role="treeitem">子节点</li>
    </ul>
  </li>
</ul>

<!-- 加载状态 -->
<div role="status" aria-live="polite">加载中...</div>
```
