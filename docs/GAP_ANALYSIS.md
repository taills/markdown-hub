## A. 已实现的功能 (PRD有, 代码已实现)

### 认证与用户
- 用户注册（用户名/邮箱/密码，3-50字符，唯一性校验）
- 用户登录（用户名+密码，JWT Token认证）
- JWT Token有效期管理
- 首次注册用户自动成为管理员（第一个用户）
- 个人中心：修改密码
- 个人中心：设置偏好语言（zh-CN/zh-TW/en，实时生效）
- bcrypt密码加密存储

### 工作空间
- 创建/编辑/删除工作空间
- 工作空间公开/私有状态切换
- 工作空间拖拽排序
- 添加/移除工作空间成员
- 设置成员权限级别（read/edit/manage）
- 查看工作空间成员列表
- 工作空间级别附件上传

### 文档管理
- 文档 CRUD（创建/读取/更新/删除）
- 文档公开/私有状态切换
- 文档拖拽排序
- 文档标题编辑
- Markdown内容编辑（纯文本编辑区）
- 实时预览（HTML渲染）
- 光标同步（编辑器到预览区）
- 自动保存（debounce 500ms WebSocket同步）
- 标题提取（ParseHeadings生成锚点）

### 实时协作
- WebSocket连接管理
- 内容同步（行级差异同步，LinePatch）
- 协作指示（显示在线用户数）
- 冲突处理（服务端按时间戳合并）

### 权限系统
- 三级权限体系（read/edit/manage）
- 权限层级检查（管理员 > 所有者 > 工作空间成员 > 文档授权 > 标题级）
- 文档级直接授权（通过用户名添加协作者）
- 标题级精细权限（基于heading_anchor的字节范围控制）
- 标题级权限验证（ValidateHeadingEdits）
- 权限叠加规则（多级别取最高）

### 版本控制（快照）
- 启发式快照触发（行数变化>20/字节变化>2048/时间间隔>5分钟）
- 手动创建快照
- 列出历史快照（分页）
- 预览快照内容
- 恢复快照（pre-restore自动保存当前状态）
- 快照对比（line-level diff算法）

### 附件管理
- 文件上传（multipart/form-data）
- 工作空间级别附件存储
- 文档级别附件上传
- 粘贴图片上传（useImagePaste hook）
- 附件列表查看
- 附件下载
- 附件删除
- 未引用附件检测（parseMarkdownReferences）
- 附件跨文档引用关系（CreateReference）

### 内容导入
- URL导入（自动抓取HTML转Markdown）
- HTML内容直接导入
- 图片自动下载并重新上传为附件
- 相对URL自动处理

### 搜索功能
- 公开文档全文搜索（未登录用户）
- 用户可访问文档搜索（登录用户）
- 搜索结果高亮匹配关键词
- 搜索结果摘要提取
- 键盘上下导航（搜索结果列表）
- Cmd+K 快捷键聚焦搜索

### 公开主页
- Hero区（站点名称、副标题）
- 公开工作空间卡片网格
- 公开文档列表（含标题、摘要、阅读时长）
- 博客风格阅读体验

### 管理员后台
- 用户管理列表（分页）
- 设置/取消管理员
- 软删除用户
- 恢复用户
- 重置用户密码（随机密码生成）
- 修改用户邮箱
- 操作日志审计（分页，Action筛选）
- 站点标题配置
- 站点标题前端显示

### 多语言支持
- 前端界面三语言（zh-CN/zh-TW/en）
- 语言切换实时生效（无需刷新）
- i18n完整翻译覆盖

### 其他
- WebSocket重连机制
- 错误提示与处理（ErrorModal）
- 公开文档只读视图（/documents/:id/view）
- 公开工作空间视图（/workspaces/:id/view）
- 用户头像上传（接口支持，UI待实现）

## B. 未实现的功能 (PRD有, 代码未实现)

### P0 - 核心缺失功能

| 功能 | PRD描述 | 说明 |
|------|---------|------|
| **评论功能** | 文档级/章节级Markdown评论，支持编辑/删除 | 代码中无CommentService，无comments表操作，无评论相关API handler |
| **第三方登录** | 钉钉/企业微信/飞书扫码登录，绑定/解绑 | social_accounts表存在但代码无任何social登录逻辑 |
| **AI辅助功能** | Ask AI智能问答/内容总结/缩写/扩写/补全，对话历史 | 代码中无AIService，无ai_conversations/ai_messages表操作 |
| **PDF导出** | 文档导出PDF，保留样式 | 无export handler，无PDF生成逻辑 |
| **ZIP批量导出** | PDF+附件打包下载 | 无export handler |
| **节点可见性控制** | public/internal双模式+继承控制 | models.Document只有is_public布尔值，无visibility字段，无inherit_visibility逻辑 |
| **节点权限一键清除** | 清除节点所有权限设置+可见性，恢复继承 | 无ClearPermissions相关逻辑 |

| 权限系统 | 75% | 缺少角色/部门授权继承、节点权限一键清除 |
| 版本控制 | 95% | 快照系统完整，Diff算法健壮 |
| 附件管理 | 85% | 缺少引用统计、引用列表UI |

| 公开主页 | 90% | UI完整，缺少深色模式 |
| 管理员后台 | 95% | 功能完整，少量边界情况待完善 |
| 多语言 | 90% | 前端完整，后端i18n API未实现 |

### Bug风险评估

| 风险点 | 级别 | 说明 |
|--------|------|------|
| 标题权限字节范围计算 | 中 | ParseHeadings字节范围在内容频繁变更时可能漂移 |
| WebSocket重连风暴 | 中 | 断线重连无指数退避，高并发可能打爆服务器 |
| 快照触发启发式阈值 | 低 | 现有阈值（20行/2048字节/5分钟）合理但未经大数据验证 |
| 权限叠加规则 | 低 | 多级别取最高逻辑清晰，但PRD描述"叠加"可能有歧义 |
| 并发文档冲突 | 低 | 服务端时间戳合并在极端并发下可能丢内容 |

### 架构质量

- **分层清晰**：Handler → Service → Store(sqlc)，依赖单向
- **错误处理**：自定义错误类型（ErrUnauthorized/ErrForbidden/ErrInvalidInput）
- **事务支持**：关键操作使用WithTransaction（快照恢复、用户注册）
- **代码复用**：storeDocToModel等转换函数避免重复逻辑
- **安全**：bcrypt密码哈希、SQL参数化查询、JWT认证中间件

### 关键待办

1. **[P0] 评论功能** - 数据模型已定义，需新建CommentService及API handler
2. **[P0] 第三方登录** - 需实现钉钉/企微/飞书OAuth流程及social_accounts管理
3. **[P0] 节点可见性** - 需扩展Document模型visibility字段及继承逻辑
4. **[P0] AI辅助** - 需新建AIService对接OpenAI兼容接口
5. **[P0] PDF/ZIP导出** - 需集成pdf生成库和zip打包逻辑
6. **[P1] 部门/角色** - 需新建DepartmentService/RoleService及管理API
7. **[P1] 权限继承控制** - document_permissions.inherit_to_children字段已建但逻辑未实现
