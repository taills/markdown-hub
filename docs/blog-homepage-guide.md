# 博客风格首页使用指南

## 🎯 功能概述

MarkdownHub 现在拥有一个现代化的博客风格首页,支持:
- 工作空间以"栏目"形式展示
- 公开文档以"文章"形式展示,包含标题、时间、摘要(前200字)和阅读时长
- 响应式设计,完美适配移动端和桌面端
- 丰富的微交互和动画效果

## 📸 页面预览

### Hero 区域
- 大标题显示站点名称(可在管理后台设置)
- 副标题:"知识分享 · 协作写作 · Markdown创作平台"
- 根据用户登录状态显示不同按钮:
  - 未登录: "登录" + "开始使用"
  - 已登录: "进入编辑器"

### 栏目区 (工作空间)
- 网格布局展示公开的工作空间
- 每个卡片包含:
  - 渐变色图标
  - 工作空间名称
  - 最后更新时间
- Hover 效果: 卡片上浮,边框高亮,顶部渐变条出现

### 文章区 (公开文档)
- 列表式布局展示公开文档
- 每篇文章显示:
  - 文章标题
  - 发布/更新时间 + 预计阅读时长
  - 内容摘要(前 200 字,自动去除 Markdown 语法)
  - "阅读全文"链接
- Hover 效果: 卡片上浮,阴影增强,标题变色

## 🔧 如何使用

### 1. 设置公开内容

#### 设置公开工作空间
1. 登录 MarkdownHub
2. 进入"文档列表"页面
3. 点击工作空间右侧的"设置"按钮
4. 勾选"公开"选项
5. 保存后,该工作空间将在首页"栏目"区域显示

#### 设置公开文档
1. 在文档编辑器中打开任意文档
2. 点击顶部工具栏的"公开/私有"切换按钮
3. 设为"公开"后,文档将在首页"最新文章"区域显示

### 2. 自定义站点标题
1. 以管理员身份登录
2. 进入"管理" → "站点设置"
3. 修改"站点标题"字段
4. 保存后,首页 Hero 区域的大标题将更新

### 3. 访问首页
- 未登录用户访问 `/` 或 `/home` 路径
- 已登录用户访问 `/home` 路径
- 已登录用户访问 `/` 会自动跳转到 `/documents`

## 🎨 设计亮点

### 视觉设计
- **渐变色背景**: Hero 区使用紫色渐变 (`#667eea` → `#764ba2`)
- **卡片设计**: 白色背景,圆角,微妙阴影
- **图标系统**: SVG 图标配合渐变色背景
- **字体层级**: 清晰的字号和字重区分

### 交互设计
- **Hover 反馈**: 所有可点击元素都有明确的悬停效果
- **平滑动画**: 使用 `cubic-bezier` 缓动函数
- **视觉引导**: 箭头图标引导用户点击"阅读全文"

### 响应式设计
- **移动端**: 单列布局,减小字号和间距
- **平板**: 栏目 2 列,文章单列
- **桌面**: 栏目自适应 3+ 列,文章单列

## 📊 内容展示逻辑

### 摘要生成
- 自动提取文档内容前 200 字符
- 去除 Markdown 语法符号 (`#`, `*`, `` ` ``, `[]`, `()`)
- 超过 200 字自动添加省略号 `...`

### 阅读时长计算
- 基于平均阅读速度 400 字/分钟
- 公式: `Math.max(1, Math.ceil(字符数 / 400))`
- 最少显示 1 分钟

### 排序规则
- 工作空间: 按 `updated_at` 降序排列
- 文档: 按 `updated_at` 降序排列(最新更新的在最前)

## 🌐 多语言支持

首页支持三种语言:
- **简体中文** (zh-CN)
- **繁体中文** (zh-TW)
- **英文** (en)

语言会根据用户浏览器设置或个人中心设置自动切换。

## 💡 最佳实践

### 内容创作建议
1. **文档标题**: 清晰、简洁、有吸引力
2. **前 200 字**: 确保前 200 字能够完整表达文章主旨
3. **段落结构**: 使用清晰的段落和标题层级
4. **更新频率**: 定期更新内容保持首页新鲜度

### 工作空间管理
1. **命名规范**: 使用简短、描述性的名称(如"技术博客"、"产品文档")
2. **内容分类**: 按主题或类别组织工作空间
3. **公开选择**: 只公开高质量、完善的内容

## 🔍 常见问题

### Q: 为什么我的文档没有显示在首页?
A: 请检查:
1. 文档是否设置为"公开"状态
2. 文档所在的工作空间是否有公开权限
3. 刷新页面查看最新状态

### Q: 如何隐藏某个文档但保持工作空间公开?
A: 进入文档编辑器,将该文档切换为"私有"状态即可。

### Q: 摘要显示的内容不理想怎么办?
A: 摘要是自动提取文档前 200 字生成的,建议:
1. 调整文档开头的内容
2. 确保开头段落能清晰表达文章主旨
3. 避免在开头使用大量 Markdown 语法

### Q: 可以修改摘要长度吗?
A: 当前摘要长度固定为 200 字符。如需修改,可以编辑 `web/src/components/HomePage.tsx` 中的 `getExcerpt` 函数。

### Q: 首页加载慢怎么办?
A: 首页性能已经过优化,如遇加载慢:
1. 检查网络连接
2. 减少公开文档的数量
3. 考虑启用 CDN 加速静态资源

## 🚀 进阶定制

### 修改配色方案
编辑 `web/src/styles.css` 中的 `.blog-hero` 样式:
```css
.blog-hero {
  background: linear-gradient(135deg, #你的颜色1, #你的颜色2);
}
```

### 调整卡片布局
修改 `.blog-categories` 的 `grid-template-columns`:
```css
.blog-categories {
  grid-template-columns: repeat(auto-fill, minmax(320px, 1fr));
}
```

### 自定义摘要长度
编辑 `HomePage.tsx` 中的 `getExcerpt` 函数:
```typescript
const getExcerpt = (content: string, maxLength: number = 300) => {
  // 修改 maxLength 参数即可
}
```
